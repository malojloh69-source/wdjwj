# OTS FanPay Mini App v3

Telegram Mini App с полностью обновлённым выводом, реальным пополнением Telegram Stars, TON Connect, мультивалютными балансами и сделками по шестизначному коду.

## Что изменено

### Новый дизайн вывода

- единая минималистичная карточка операции;
- текущий доступный баланс;
- разные реквизиты для карты, криптокошелька и Telegram username;
- быстрый выбор 25%, 50%, 75% и MAX;
- расчёт суммы к получению;
- мягкие тени, аккуратные границы и адаптация под светлую/тёмные темы;
- заявка на вывод создаётся на сервере, а сумма сразу резервируется из внутреннего баланса.

Важно: сервер создаёт заявку со статусом `pending`, но автоматическая отправка денег, криптовалюты или Stars получателю не реализована. Для настоящих выплат нужен отдельный платёжный модуль и административная обработка.

### Настоящее пополнение Telegram Stars

Пополнение Stars работает через официальный Bot API:

1. Mini App отправляет сумму на `/api/stars/invoice`.
2. Сервер проверяет `Telegram.WebApp.initData`.
3. Бот создаёт настоящий invoice в валюте `XTR` через `createInvoiceLink`.
4. Mini App открывает его методом `Telegram.WebApp.openInvoice`.
5. Бот подтверждает `pre_checkout_query`.
6. После `successful_payment` сервер сохраняет `telegram_payment_charge_id` и зачисляет Stars на внутренний баланс пользователя.
7. Повторное зачисление одной оплаты блокируется по `telegram_payment_charge_id`.
8. Каждое зачисление записывается в серверный журнал `balanceEvents`.

Официальная документация:

- https://core.telegram.org/bots/payments-stars
- https://core.telegram.org/bots/api#createinvoicelink
- https://core.telegram.org/bots/webapps#validating-data-received-via-the-mini-app

### Балансы в профиле

В профиле отображаются все валюты проекта:

`TON`, `USDT`, `STARS`, `RUB`, `KGS`, `EUR`, `GBP`, `JPY`, `TRY`, `UAH`, `KZT`, `BTC`, `ETH`, `CNY`.

Баланс Stars изменяется только после подтверждённой оплаты ботом. Для автоматического зачисления TON, USDT и остальных валют нужно подключить отдельные серверные обработчики блокчейна/платёжных систем.

### Сделки по коду

Сделки вынесены в облачную Supabase Edge Function. Создание, подключение по коду, статусы и история работают без локального `npm start`. Создатель может скопировать готовую ссылку; при её открытии код подставляется автоматически.


- каждой новой сделке сервер выдаёт уникальный шестизначный код;
- второй пользователь может нажать «Войти по коду» и присоединиться;
- активный код нельзя занять третьему участнику;
- статусы идут последовательно:
  - `unpaid` — «Не оплачено»;
  - `paid` — «Оплачено»;
  - `completed` — «Выполнено»;
- выполненные сделки переносятся в историю.

Статусы являются рабочим процессом сделки. Они сами по себе не подтверждают банковский или блокчейн-платёж и не заменяют эскроу.

### Поле промокода

В профиль добавлено оформленное поле ввода промокода. Активных промокодов в проекте нет: при отправке любого значения интерфейс сообщает, что код не найден. Никакие права или баланс через это поле не выдаются.

## Структура

- `index.html` — интерфейс Mini App;
- `app.js` — кошельки, Stars, профили и сделки;
- `styles.css` — основной дизайн;
- `app-config.js` — адрес API;
- `wallet-config.js` — TON Connect и адрес получения TON/USDT;
- `tonconnect-manifest.json` — TON Connect manifest;
- `bot/server.js` — статический сервер, Bot API, Stars, балансы, сделки и выводы;
- `bot/data/store.json` — создаётся автоматически при запуске и хранит данные прототипа;
- `balanceEvents` внутри хранилища — журнал зачислений Stars и резервирования средств при выводе.

## Настройка

### 1. Telegram-бот

Создайте бота через `@BotFather` и получите токен.

Скопируйте переменные из `bot/.env.example` в настройки хостинга:

```env
BOT_TOKEN=123456789:replace_with_bot_token
MINI_APP_URL=https://your-domain.example
PORT=8080
HOST=0.0.0.0
TELEGRAM_UPDATES_MODE=polling
WEBHOOK_SECRET=replace_with_long_random_secret
ALLOW_DEMO=false
MAX_INIT_DATA_AGE_SECONDS=86400
SUPPORT_USERNAME=your_support_username
TERMS_URL=https://your-domain.example/terms
```

Файл `.env` автоматически не загружается, чтобы проект оставался без внешних зависимостей. Переменные нужно передать через панель хостинга, Docker, systemd или командную строку.

### 2. Режим получения обновлений

Для обычного постоянно работающего Node.js-сервера:

```env
TELEGRAM_UPDATES_MODE=polling
```

Сервер отключит старый webhook и запустит `getUpdates`.

Для публичного HTTPS-сервера с webhook:

```env
TELEGRAM_UPDATES_MODE=webhook
WEBHOOK_SECRET=длинная_случайная_строка
MINI_APP_URL=https://your-domain.example
```

Сервер также настраивает команды `/start`, `/paysupport`, `/terms` и кнопку меню Mini App.

Webhook будет настроен на:

```text
https://your-domain.example/api/telegram/webhook
```

Не запускайте одновременно несколько экземпляров polling-сервера с одним токеном.

### 3. TON Connect

В `wallet-config.js` укажите публичный TON-адрес получателя:

```js
window.OTS_WALLET_CONFIG = {
  manifestUrl: new URL('./tonconnect-manifest.json', window.location.href).href,
  network: '-239',
  recipientAddress: 'EQ...ВАШ_АДРЕС...',
  usdtJettonMaster: 'EQCxE6mUtQJKFnGfaROTKOt1lZbDiiX1kCixRv7Nw2Id_sDs',
};
```

В `tonconnect-manifest.json` замените тестовый домен на настоящий HTTPS-домен.

### 4. API

Если интерфейс и сервер размещены на одном домене, оставьте в `app-config.js`:

```js
window.OTS_APP_CONFIG = { apiBase: '' };
```

При отдельном API укажите его HTTPS-адрес и настройте CORS на сервере самостоятельно.

### 5. Что обязательно заполнить перед запуском

- `BOT_TOKEN` — токен именно того бота, которому должны поступать Stars;
- `MINI_APP_URL` — публичный HTTPS-адрес приложения;
- `SUPPORT_USERNAME` — username поддержки без `@`;
- `TERMS_URL` — публичная страница условий;
- `wallet-config.js` и `tonconnect-manifest.json` — адреса для TON Connect.

Не размещайте `BOT_TOKEN` в `app.js`, HTML или любом другом файле фронтенда.

## Размещение

Для создания, подключения и изменения статусов сделок локальный Node.js не нужен. Настройте Supabase по инструкции `SETUP_CLOUD_DEALS.md`, после чего разместите HTML/CSS/JS на любом HTTPS-хостинге.

Node.js остаётся отдельной необязательной частью только для уже реализованных функций Stars, серверных балансов и заявок на вывод. Если они тоже нужны, сервер можно запускать на хостинге командой:

```bash
npm start
```

Для временной локальной проверки серверной части можно установить `ALLOW_DEMO=true`, но рабочие облачные сделки проверяют пользователя по подписанному `Telegram.WebApp.initData`.

## Перед продакшеном

Текущий JSON-файл подходит для прототипа и одного Node.js-процесса. Для реального сервиса замените его на PostgreSQL или другую транзакционную базу данных, добавьте:

- административную панель заявок на вывод;
- журнал изменения баланса;
- роли и права сотрудников;
- резервное копирование;
- rate limiting;
- мониторинг платежей;
- проверку TON/USDT транзакций на сервере;
- настоящий эскроу для сделок;
- обработку возвратов Stars через `refundStarPayment`;
- полноценную поддержку команды `/paysupport`.


## Облачные сделки без локального запуска

Пошаговая настройка находится в `SETUP_CLOUD_DEALS.md`. После заполнения `dealApiUrl` пользователям не нужно запускать Node.js. Локальный сервер остаётся только для функций Stars, балансов и выводов, пока они не перенесены в облачные функции.
